### 3.2 游戏算法系统

#### 3.2.1 NatureAlg (自然算法)

**功能**: 生成随机转轮结果

**核心结构**:
```c
typedef struct {
    int32_t** RollTable;      // 5个转轮的符号表
    int32_t RollTableSizes[5]; // 每个转轮的大小
} NatureAlg;
```

**工作流程**:
1. **初始化**: `NatureAlg_Init()` - 预定义5个转轮的符号分布
2. **生成矩阵**: `NatureAlg_GenRndMxu()` - 从转轮表中随机选择符号
3. **结果计算**: `Matrix_u_computerMatrix()` - 计算中奖结果

**转轮配置**:
- 5个转轮（Reel 0-4）
- 每个转轮包含不同数量的符号（60-71个符号）
- 符号类型：0-11（包括普通符号、Wild、Scatter、Bonus）

#### 3.2.2 游戏结果类型

**ResultType_e**:
- `RT_Lose`: 失败（无中奖）
- `RT_Win`: 普通胜利（中奖线）
- `RT_FreeWin`: 免费游戏（3+ Scatter触发）
- `RT_BonusWin`: 奖池游戏（3+ Bonus触发）

**OpenType_e**:
- `OT_Normal`: 正常开启
- `OT_Give`: 赠送开启（免费游戏回合）

### 3.3 游戏结果生成流程

**核心函数**: `GetNormalResult()`

**流程**:
```
1. 判断开启类型（OT_Normal / OT_Give）
   ├─ OT_Give: 使用已存储的免费游戏数据
   └─ OT_Normal: 生成新的游戏结果
       ├─ 生成随机矩阵（NatureAlg_GenRndMxu）
       ├─ 计算中奖线（Matrix_u_computerMatrix）
       └─ 判断结果类型
           ├─ RT_Lose: 无中奖
           ├─ RT_Win: 普通中奖
           ├─ RT_FreeWin: 触发免费游戏
           │   └─ 生成所有免费游戏回合的矩阵
           └─ RT_BonusWin: 触发奖池游戏
```


### 3.4 奖池管理系统 (LotteryManager)

**功能**: 管理3个独立的奖池（JP1, JP2, JP3）

**核心结构**:
```c
typedef struct LotteryManager {
    int32_t mTotalPlayTime;   // 总游戏次数
    int32_t mTotalPlay;       // 总投注值
    int32_t mTotalDraw;       // 总抽取值
    Lottery* mLotterys[3];    // 3个奖池实例
    int32_t mFrozenTime;      // 冻结时间
    int32_t mScale;           // 放大倍数比例（100倍）
} LotteryManager;
```

**工作流程**:
1. **初始化**: 创建3个奖池，设置基础值和最大值
2. **投注处理**: `LotteryManager_OnPlay()` - 累计投注值
3. **触发检查**: `LotteryManager_TryGetLottery()` - 根据投注值计算触发概率
4. **奖池更新**: 根据触发结果更新奖池值

**奖池配置**:
- JP1: 基础值 1×scale, 最大值 JPWeight[1]×unit×scale
- JP2: 基础值 2×scale, 最大值 JPWeight[2]×unit×scale
- JP3: 基础值 4×scale, 最大值 JPWeight[3]×unit×scale

### 3.7 调试和统计系统

**DebugInfo 结构**:
- `dwTotalPlayTime`: 总游戏次数
- `dwLooseTime`: 失败次数
- `dwNormalWinTime`: 普通胜利次数
- `dwFreeGameTime`: 免费游戏次数
- `dwBonusTime`: 奖池游戏次数
- `dwNormalWinTotalBet`: 普通胜利总倍数
- `dwFreeGameTotalBet`: 免费游戏总倍数
- `dwBonusGameTotalBet`: 奖池游戏总倍数

**RTP计算**:
```c
BaseRTP = dwNormalWinTotalBet / TotalBet
FreeRTP = dwFreeGameTotalBet / TotalBet
BounsRTP = dwBonusGameTotalBet / TotalBet
TotalRTP = BaseRTP + FreeRTP + BounsRTP
```

### 5.1 优点
1. **结构清晰**: 模块划分明确，职责分明
2. **跨平台**: 支持Windows和嵌入式平台
3. **可配置**: 通过宏定义灵活配置功能

### 5.2 需要改进的地方
1. **注释**: 部分中文注释存在编码问题（乱码）
2. **错误处理**: 部分函数缺少返回值检查
3. **内存管理**: 
   - 测试代码中使用 `malloc` 但未看到对应的 `free`
   - 全局变量管理需要改进
4. **代码规范**: 
   - 部分变量命名不够规范（如 `jpvaule` 应为 `jpvalue`）
   - `main.c` 第23行：`#include"Control/NatureAlg.h"` 缺少空格
5. **单例限制**: 使用全局变量，只支持单一游戏实例

### 5.3 潜在问题
1. **内存泄漏风险**: 
   - `main.c` 中测试代码分配的内存可能未释放
   - `GetNormalResult` 中分配了 `RoundInfo_t` 和 `Matrix_u`，需要确保释放
2. **线程安全**: 全局变量在多线程环境下不安全
3. **硬编码**: 部分配置值硬编码在代码中（如转轮表）

## 6. 关键数据流

### 6.1 游戏流程
```
玩家投注 
  → DLL_GetGameResult() 
  → GetNormalResult() 
  → NatureAlg_GenRndMxu() (生成随机矩阵)
  → Matrix_u_computerMatrix() (计算中奖)
  → 返回结果(OutResult_t) 
  → 更新玩家数据 
  → 更新调试信息
```

### 6.2 奖池流程
```
投注 
  → LotteryManager_OnPlay() (累计投注值)
  → LotteryManager_TryGetLottery() (检查触发条件)
  → 返回奖池类型和值
```

### 6.3 免费游戏流程
```
触发免费游戏 (RT_FreeWin)
  → 生成所有免费游戏回合的矩阵
  → 存储到 FreeGameInfo_t
  → 后续回合使用 OT_Give 类型
  → 从存储的数据中获取结果
```

### 6.4 通信流程
```
外部命令 
  → idea_comm 接收 
  → 编解码处理 (idea_coder)
  → 更新配置(comm_data) 
  → 通知游戏算法
```
